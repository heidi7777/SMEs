# SMEs产品需求文档 (PRD)：设计密集型中小企业 AI 业务工作台 (MVP 版) 

**文档版本**：v1.0.0 (Final) **当前阶段**：MVP 研发交付 

一、 产品定义 

本产品是一款“外白内专”的企业级垂直 AI 生产力工作台 。

系统摒弃传统大模型的“开放式闲聊”形态，通过“多模块专家分身 + 结构化选项芯片（Chips） + 资产沉淀”的链路，将企业内高薪专家的隐性经验（商业模型、高阶 Prompt）转化为全员可低门槛调用的数字化系统 。旨在解决设计型中小企业在创意孵化、视觉工程转译、客户沟通与商业包装中的效率瓶颈 。

二、 用户定义 

* 
**目标人群**：设计密集型中小型企业的初中级员工（助理设计师、前端商务/AE、新媒体运营）及设计/策略总监 。


* **用户特征与痛点**：
* 懂基础美感与业务，但**缺乏系统性商业逻辑与抽象表达能力** 。


* 
**存在严重的“冷启动障碍”**，面对空白输入框不知道如何提问 。


* 
**无法撰写高颗粒度的专业提示词（Prompt）**，跨模态（如文生图）工程能力弱 。


* 沟通中极易受甲方情绪影响，缺乏边界防御与谈判技巧 。





三、 产品边界 (Scope Definition) 

为了确保 MVP 版本的敏捷落地与核心价值验证，严格界定以下开发边界 ：

| 维度 | 包含范围内 (In-Scope) | 排除范围外 (Out-of-Scope) |
| --- | --- | --- |
| **功能领域** | 4 个高度垂直的独立业务工作台；全局资产库管理与调用 。

 | 开放式非业务闲聊；法务/行政等非核心流；跨房间自动化数据穿透 。

 |
| **交互与 UI** | Dashboard 首页指令分发；意图雷达选项拦截；滑动窗口历史截取；全局资产 `/` 快捷键唤出与表单拼装 。

 | 复杂的富文本编辑器；多端适配（移动端 H5/小程序暂不做，仅锁死 PC 网页端） 。

四、 需求定义：核心功能与底层框架 

系统划分为四个独立的业务模块与一个全局资产中枢 。每个功能点必须严格基于指定的专业框架运行 。

1. 【创意工坊】 (聚焦灵感爆发与市场洞察) 

* 
**专家人设**：产品创新总监 


* **核心功能**：
* 
**灵感生成器**：基于 **SCAMPER 创新法** 。引导输入基础品类后，强制从“替代、合并、改造等”七个维度，一键输出 3-5 个跨界创意草案 。


* 
**竞品分析**：基于 **JTBD 模型 (待办任务理论)** 。剥离竞品营销修辞，直接洞察“目标消费者究竟想雇佣该竞品完成什么隐性任务”，推导我方差异化切入点 。





2. 【视觉车间】 (聚焦电商与新媒体落地呈现) 

* 
**专家人设**：技术美术专家 (TA) 与 资深电商视觉设计 


* **核心功能**：
* 
**营销海报生图引擎**：基于 **电商场景化搭棚公式** 。套用 `[产品主体]+[陈列台面]+[环境氛围]+[虚化前景]+[光影]`，输出高转化率生图 Prompt 。


* 
**留白构图控制器**：基于 **视觉引导法则** 。针对排版加字需求，在 Prompt 中强制注入“三分法则”、“右侧 30% 负空间”等构图参数 。


* 
**材质与渲染参数转译器**：基于 **PBR 材质物理渲染解构流** 。将模糊的“高级感”大白话精准翻译为包含基础色、粗糙度、金属度及渲染器（如 Octane）的高颗粒度代码段 。





3. 【沟通台】 (聚焦情绪脱敏与边界防御) 

* 
**专家人设**：资深客户总监 


* **核心功能**：
* 
**需求提炼**：基于 **FIRE 模型 (事实-解释-反应-结果)** 。过滤甲方长段反馈中的情绪词与指责，仅向设计端输出客观的《待改需求事实清单》 。


* 
**需求确认**：基于 **冰山模型** 。挖掘甲方“要网感”等模糊词汇下的真实商业动机，翻译为量化视觉参数，生成 A/B 方案供客户做封闭式选择 。


* 
**沟通方案**：基于 **非暴力沟通 (NVC) 与 BATNA 谈判底线** 。针对客户超纲要求，生成“陈述事实 ➔ 共情 ➔ 加价/延期备选方案”的话术 。





4. 【营销工坊】 (聚焦高溢价商业包装) 

* 
**专家人设**：营销与策略总监 


* **核心功能**：
* 
**提案 PPT 框架生成器**：基于 **黄金圈法则 (Why-How-What)** 。将设计升维包装成“为何设计 ➔ 如何实现 ➔ 最终效果”的高溢价商业汇报逐字稿 。


* 
**营销文案库**：基于 **STP 细分定位策略** 。强制对齐目标人群，裂变带网感的小红书种草文案或电商卖点清单 。





5. 【全局资产库】 (知识沉淀中枢) 

* 
**核心功能**：跨房间的提示词模板、高阶话术管理 。支持网格化文件夹分类存储，保存时支持设置 `[变量占位符]` 。



五、 需求定义：前端交互与状态流转 

1. 全局布局与状态指示 

* 
**侧边栏定位**：左侧导航栏常驻四大模块与资产库 。当前所处房间的菜单项必须呈现明显的高亮选中态（如背景变灰并带竖线），其余暗色 。


* 
**工作台首页 (Dashboard)**：未进入房间前，右侧主视觉**不展示目录大入口** 。


* 
**中部**：横向排列 3 张带阴影的【破冰快捷指令卡片】，点击直接携带指令跳转对应房间并新建会话 。


* 
**底部**：展示全局最近会话列表，点击直接穿透加载对应聊天记录 。





2. 房间内缺省与历史管理 

* 
**首次进入空状态**：无历史记录时，屏幕中央仅渲染轻量级的悬浮引导字样（如：“试试直接复制粘贴客户的长篇修改建议”） 。用户在底部输入框一旦开始打字，悬浮字样立即消失 。


* 
**历史状态**：存在记录时，房间默认展示当前模块的【历史会话列表】 。


* 
**自动命名与清理**：系统自动截取用户首条消息的前 15 个字符作为会话标题（超出用 `...` 省略） 。历史列表项支持鼠标悬停显示【🗑️ 删除】图标 。



3. 核心对话流与输入规范 

* 
**键盘与限制**：输入上限为 **1500 字**，超出防抖标红 。按 `Enter` 发送消息，按 `Shift + Enter` 文本换行 。


* 
**意图拦截 (Chips)**：系统抛出动态选项芯片时 ：


* 
**单选即发**：用户单击任意 Chip，该文本即刻作为新输入发送，不设多选或二次确认 。


* 
**强制覆盖**：用户无视 Chips 直接打字发送，前端立即接收新输入，覆盖旧状态 。




* 
**兜底机制** ：


* 
**空发兜底**：直接发送空内容不报错，系统自动回复专业引导话术并附带 3 个盲猜 Chips 。


* 
**网络异常**：API 超时则在流中渲染红色气泡“网络或服务响应超时，请重试” 。




* 
**气泡操作**：每个 AI 生成的文本气泡底部，常驻 **【📋 复制】**（点击后文案变为“已复制”2秒）与 **【📌 收藏】** 按钮 。



4. AI 响应与视线追踪 

* 
**加载状态**：请求期间底部输入框置灰禁用 。对话流中禁止使用传统旋转 Loading，必须使用具有呼吸动画的**骨架屏 (Skeleton)** 占位 。


* 
**自动滚动**：消息发送或 AI 渲染时，滚动条平滑自动触底 。**打断机制**：若检测到用户手动向上滚动查看历史，自动触底即刻暂停 。


* 
**滑动窗口机制**：向大模型发起请求时，前端强制仅截取当前会话**最近 10 条消息（5轮） + 最新输入**组装 Payload，严禁全量发送 。



5. 资产调用链路 (Modal 交互) 

* 
**触发**：在底部输入框敲击键盘 `/`，上方悬浮弹出资产候选列表 。


* 
**弹窗拼装**：选中带占位符的配方后，纯净的【配方配置 Modal】居中弹出，展示变量填空框 。


* 
**打断与确认** ：


* 用户点击右上角 `[ X ]` 或外部黑色遮罩层 (Mask) 即可关闭弹窗并退回输入框 。


* 用户填完变量点击确认，后台隐式拼装完整 Prompt 并直接发起大模型请求 。





六、 边缘情况处理 

* 
**AI 响应等待** ➔ 输入框置灰禁用防抖，对话流展示呼吸骨架屏 (Skeleton)，取代传统转圈 Loading 。


* 
**首次进入/无历史记录** ➔ 首页展示 3 张破冰快捷卡片直接发指令；进入房间后中央悬浮轻量引导字样（打字即消失） 。


* 
**直接发送空内容** ➔ 严禁报错，系统自动回复专业引导话术并附带 3 个盲猜 Chips 。


* 
**网络异常/API 超时** ➔ 对话流中渲染红色错误气泡：“网络或服务响应超时，请重试” 。


* 
**历史会话命名激增** ➔ 截取首条消息前 15 字符自动命名，超出用 `...` 截断 。


* 
**超长文本视线追踪** ➔ 消息发送/渲染时平滑自动触底；若检测到用户手动上滑查看历史，立即打断自动滚动 。


* 
**中断资产调用** ➔ 唤起变量填空 Modal 后，点击 `[X]` 或黑色遮罩层 (Mask) 立即关闭并退回输入状态 。


* 
**刷新/关闭页面** ➔ 数据不丢失，所有状态实时写入 LocalStorage 。
